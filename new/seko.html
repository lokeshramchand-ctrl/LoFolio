System Architecture

Event-Driven Logic

Seko isn't just a CRUD app. It utilizes a microservices approach where the Order Service, Inventory Service, and Notification Engine communicate asynchronously via RabbitMQ.

This ensures that when a user places an order, the inventory is locked immediately in Redis, preventing overselling even during high-traffic spikes.

Engineering Challenges

The "Thundering Herd"

When a flash sale goes live, thousands of users hit the API simultaneously. We implemented connection pooling in Postgres and aggressive caching strategies in Redis to handle 10k+ concurrent requests without DB locking.

Geo-Spatial Search

Delivering in 15 minutes requires precise location data. We used PostGIS to query available sellers within a 2km radius of the user's lat/long, filtering results in under 20ms based on the Haversine formula.

State Management

Using Flutter BLoC pattern, we separated business logic from UI. This ensured that the cart state persisted accurately across the app, even if the user lost internet connectivity momentarily.

Real-time Order Tracking

Integrated WebSockets to push order status updates (Packed, Shipped, Delivered) instantly to the client, removing the need for battery-draining polling mechanisms.
