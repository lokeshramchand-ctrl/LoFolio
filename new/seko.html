SEKO

SEKO is a hyperlocal commerce engine designed to revive community supermarkets in gated societies. As quick-commerce apps grew, residents began ordering from platforms like Zepto despite having fully stocked supermarkets just downstairs. SEKO bridges this gap by giving local shops the same technological advantage— fast inventory sync, real-time ordering, and a seamless mobile shopping experience that keeps revenue within the community.

The Arsenal

Flutter

A unified codebase powering iOS and Android with high-performance UI rendering and persistent state handling for carts and orders.

Django & Python

A structured backend managing real-time orders, seller dashboards, pricing rules, and zone-based delivery logic through robust REST APIs.

Redis & Sockets

Handles ultra-fast stock updates so users always see live inventory, preventing overselling during peak shopping hours.

PostgreSQL

Enhanced with geo-spatial querying to instantly map users to the nearest community store inside their gated society.

System Architecture

Full-Stack Flow.

SEKO follows a layered architecture where the Flutter app communicates with a Django backend through NGINX. PostgreSQL handles persistent storage, Redis manages caching and sessions, while Celery processes background tasks asynchronously. The shopkeeper accesses an admin dashboard to manage inventory and orders in real-time.

Technical Hurdles

01

The Core Problem

Quick-commerce platforms captured the attention of residents living in gated communities, leading to a decline in purchases from their own nearby supermarkets. SEKO was engineered to restore this balance by giving local stores the same technological edge—instant inventory, rapid delivery, and mobile-first convenience—ensuring that community stores remain competitive.

02

The "Thundering Herd"

When a flash sale goes live, thousands hit the API simultaneously. We implemented connection pooling in Postgres and aggressive caching strategies in Redis to handle 10k+ concurrent requests without database locking.

03

State Management

Using the Flutter BLoC pattern, the app ensures consistent cart state across navigation, device changes, and short-term connectivity drops, preserving a reliable shopping experience.

04

Geo-Spatial Search

Delivering within minutes requires precise store-user mapping. PostGIS enables rapid geo-spatial queries to fetch stores within a 2km radius of the user's location, completing each lookup in under 20ms.
