Technical Hurdles

01

The "Thundering Herd"

When a flash sale goes live, thousands hit the API simultaneously. We implemented connection pooling in Postgres and aggressive caching strategies in Redis to handle 10k+ concurrent requests without DB locking.

02

State Management

Using the Flutter BLoC pattern, we separated business logic from UI. This ensured that the cart state persisted accurately across the app, even if the user lost internet connectivity momentarily.

03

Geo-Spatial Search

Delivering in 15 minutes requires precise location data. We used PostGIS to query available sellers within a 2km radius of the user's lat/long, filtering results in under 20ms.
