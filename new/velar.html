System Design

Async Processing

Financial data is heavy. Velar avoids UI blocking by offloading parsing tasks to a background worker.

When a bank email arrives, the Ingestion Service pushes a message to RabbitMQ. A separate Worker Node consumes the message, runs the Regex classification, and updates MongoDB—ensuring the frontend remains buttery smooth.

Key Challenges

Unstructured Data Parsing

Bank emails are inconsistent. I built a robust Regex Engine that identifies patterns in subject lines and bodies, extracting Amount, Merchant, and Date with 98% accuracy, handling edge cases like currency symbols and date formats.

Self-Learning Categorization

Instead of a heavy ML pipeline, I implemented a lightweight weighting system. If a user corrects "Uber" from 'Travel' to 'Transport', the system adjusts the category weight for that merchant globally, improving future accuracy.

Real-Time Synchronization

Using RabbitMQ, the system decouples ingestion from display. This ensures that heavy parsing jobs do not block the Node.js event loop, allowing the Flutter app to remain responsive even while processing hundreds of emails.

Automated Deployment

I containerized the backend using Docker and set up a GitHub Actions pipeline. This reduced manual deployment time by 70%, automatically running tests and pushing images to the registry on every commit.
